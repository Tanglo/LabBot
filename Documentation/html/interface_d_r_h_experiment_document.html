Detailed Description

A document object to handle the saving and opening of experiment data files.

A DRHExperimentDocument object can be used to control the saving of LabBot experiment data files in the usual NSDocument way. Similar to the NSDocument class, it is intended that DRHExperimentDocument be subclassed rather than using it out right. As for NSDocument, when you subclass DRHExperimentDocument you must override one reading and one writing method. In the simplest case, you can override the data-based reading and writing methods, readFromData:ofType:error: and dataOfType:error:. In addition you must also override newExperimenterWindowController to create and return an DRHExperimenterWindowController object to control the main document window. If you don't want to use a DRHExperimenterWindowController as you main window controller or you want to avoid the use of window controllers altogether you should subclass NSDocument instead.

Note
This class is in its infancy and as of version 0.2 of the LabBot framework the functionality of DRHExperimentDocument is basic. The only features it adds to the standard NSDocument funcationality is to store instances of DRHExperimenterWindowController and DRHSubjectWindowController and enforce and manage the creation of a single DRHExperimenterWindowController object. In many cases it will be easier to use a subclass of NSDocument.

Method Documentation

- (id) newExperimenterWindowController

It is compulsory to override this function. It should return an instance of a DRHExperimenterWindowController object that is used to control the receiver's main window.

This method is called by makeWindowControllers and the returned object is added to the receivers window controllers and set as the value for the receiver's experimentWindowController property.

Returns
An instance of DRHExperimenterWindowController that will become the window controller for the reciever's main window.
